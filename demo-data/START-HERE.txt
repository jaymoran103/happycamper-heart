=====================================================
 HappyCamper — Sample Data
=====================================================

These folders contain sample rosters for exploring HappyCamper without your own data. 
Each folder holds a matched pair of files:

  - Camper roster (User Report)
  - Activity roster (Elective Report)

To use them: open HappyCamper, select import, and choose the two files from the same folder.



-----------------------------------------------------
 1-single-cabins
-----------------------------------------------------
A single cabin with fields for all main features. 
Data has each type of issue HappyCamper is designed to detect:

- Camper with incomplete activity count
- Campers with unrequested activities
- Campers with swim level mismatches

Try isolating specific concerns using the filters sidebar.

-----------------------------------------------------
 2-full-session
-----------------------------------------------------
Real (anonymized) data from session 6, 2024. 
This session had a large number of campers, all of whom were successfully assigned 3 activities.

This pair is good for exploring:

- The preference feature: given the high volume of campers and odd camptivity lineup, there's high 
variation in preference satisfaction. To see more of stats, use the 'Column Visibility' menu in from the top bar.

- Program name standardization: Source data distinguishes between Outpost and its All-Gender section (Campminder 
necessitated this for registration purposes.) HappyCamper normalizes this, so the Outpost cohort can be treated as
a single group.

-----------------------------------------------------
 3-full-session-gaps
-----------------------------------------------------

Same data set as session 6, doctored to introduce activity gaps for happycamper to catch. Note that outpost is treated as 
a mixed session due to mismatch between camper activity counts, while the basic traditional camp program retains the
same 3 round status as before.

-----------------------------------------------------
4-assignment-iterations
-----------------------------------------------------

This set contains a series of real (anonymized) rosters generated while leaders progressively 
caught and resolved issues based 
on HappyCamper analysis and outside input. 

Camper Roster
- This report version was generated after Featherstone's preferences had been input, remedying 
the reason for missing assignments in activity-s1v1.
- Includes a staff kid, exempt from the activity system, and lacking several standard fields. This 
outlier can be filtered out using the Programs Filter.

Activity Roster
- The activity roster was generated 4 times, each of which can be applied as an input to simulate 
steps in the validation and revision process.

1. activity-s1v1-missingcabin.csv – lacks assignments for Featherstone lodge, due to lack of input preferences. Discovered while exploring HappyCamper results. Unknown at the time: this roster also lacks swim assignments
2. activity-s1v2-missingswimlessons.csv – lacks swimming assignments, discovered while exploring HappyCamper results and wondering why swimming had no assignees. Main motivator for new activity search feature. 
3. activity-s1v3-complete.csv – resulting roster once above issues were rectified
4. activity-s1v4-tuesdayswitches.csv – roster after second-day swaps were made. No monumental changes, included for posterity.
5. activity-s1v5-duplicatesdoctored – doctored version of v4, swapping some activities to demo the new duplicate detection feature.
